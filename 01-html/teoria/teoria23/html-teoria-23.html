TEORÍA 23 : Etiquetas <UL> y <LI>, para crear listas no ordenadas.

EXPLICACIÓN:

Las etiquetas etiquetas <UL> y <LI> son etiquetas en bloque.

Su cometido es estructural.

Dichas etiquetas las usaremos para crear listas no ordenadas y darlas formato.

Diremos que el objeto u objetos que esten entre las etiquetas <LI> y </LI> serán los elementos de la lista e irán englobados entre las etiquetas <UL> y </UL>.

EJEMPLO:

Ejemplo de <UL>:

Sintaxis básica:

<UL>
	<LI> Elemento 1 </LI>
	<LI> Elemento 2 </LI>
	<LI> Elemento 3 </LI>
</UL>

Esto es lo que hace el código anterior, en naranja los elementos de la lista.

  • Elemento 1
  • Elemento 2
  • Elemento 3

PROPIEDADES:

Las etiquetas <UL> y <LI> solo admiten la propiedad "TYPE" con la que cambiaremos el símbolo que marcará el elemento, en este caso las propiedades START y VALUE no serán aplicables debido a que la lista no tiene orden.

  • TYPE: Con esta propiedad podremos definir la representación grafica de los símbolos de las listas, hay 3 valores diferentes posibles, "disc" para utilizar un disco negro para señalizar los elementos, "circle" que lo convertirá en un circulo sin relleno y "square" para que su símbolo sea un cuadrado.

    • Ej. Ejemplo de lista no ordenada con valor disc.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3

    • Ej. Ejemplo de lista ordenada alfabética mayúscula.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3

    • Ej. Ejemplo de lista ordenada alfabética minúscula.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3
