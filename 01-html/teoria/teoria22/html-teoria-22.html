TEORÍA 22 : Etiquetas <OL> y <LI>, para crear listas ordenadas.

EXPLICACIÓN:

Las etiquetas etiquetas <OL> y <LI> son etiquetas en bloque.

Su cometido es estructural.

Dichas etiquetas las usaremos para crear listas ordenadas y darlas formato.

Diremos que el objeto u objetos que esten entre las etiquetas <LI> y </LI> serán los elementos de la lista e iran englobados entre las etiquetas <OL> y </OL>.

EJEMPLO:

Ejemplo de <OL>:

Sintaxis básica:

<OL>
	<LI> Elemento 1 </LI>
	<LI> Elemento 2 </LI>
	<LI> Elemento 3 </LI>
</OL>

Esto es lo que hace el código anterior, en naranja los elementos de la lista.

  1. Elemento 1
  2. Elemento 2
  3. Elemento 3

PROPIEDADES:

Las etiquetas <OL> y <LI> admiten varias propiedades que la modificarán: TYPE y START para las etiquetas de lista y VALUE para la etiqueta de elementos.

  • TYPE: Con esta propiedad podremos definir la numeración de la lista, bien sea numeros o letras y hay 5 valores diferentes posibles, "1" para listas de números, "A" para letras mayúsculas, "a" para crear listas con letras minúsculas, "I" para numeración romana mayúscula e "i" en numeros romanos en minúsculas.

    • Ej. Ejemplo de lista ordenada numérica.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3

    • Ej. Ejemplo de lista ordenada alfabética mayúscula.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3

    • Ej. Ejemplo de lista ordenada alfabética minúscula.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3

    • Ej. Ejemplo de lista ordenada romana mayúscula.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3

    • Ej. Ejemplo de lista ordenada romana minúscula.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3

  • START: Con esta propiedad podremos definir cual será el inicio de la lista y su valor será un número, de este modo si damos el valor 3 la lista empezará por el número 3, la letra C o III en caso de que la lista sea de numeración romana.

    • Ej. Ejemplo de lista ordenada numérica con inicio en el 3.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3

    • Ej. Ejemplo de lista ordenada alfabética mayúscula con inicio en la C.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3

    • Ej. Ejemplo de lista ordenada romana mayúscula con inicio en el III.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3

  • VALUE: Con esta propiedad podremos definir la numeración de un elemento de la lista al igual que con el atributo START que vimos antes, al cambiar uno de los elementos los siguientes seguiran el orden dictado por este, en el caso de que el valor de un elemnto sea 10 el resto de elementos seguirá de forma correlativa a partir de ese número aunque no le correspondiera.

    • Ej. Ejemplo de lista ordenada numérica con value 10 en el segundo elemento.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3
      4. Elemento 4
      5. Elemento 5
      6. Elemento 6

    • Ej. Ejemplo de lista ordenada numérica con value avanzando de 10 unidades por elemento.

      1. Elemento 1
      2. Elemento 2
      3. Elemento 3
      4. Elemento 4
      5. Elemento 5
      6. Elemento 6
