TEORÍA 20 : Etiquetas <ABBR> y <ACRONYM>, uso de abreviaturas y acrónimos.

EXPLICACIÓN:

Las etiquetas <ABBR> y <ACRONYM> son etiquetas en linea.

Su cometido es ornamental.

Su uso irá destinado a definir dentro de un documento las abreviaturas <ABBR> y los acronimos <ACRONYM>. La diferencia entre el uso de una etiqueta y otra es bastante sutil, para definirlo de una forma simple pondremos dos ejemplos:

Sr. es una abrebiatura de Senor mientras que ONU es el acrónimo de Organización de las Naciones Unidas.

Visualmente no podremos detectar ninguna diferencia si no insertamos ningun atributo

EJEMPLO:

Ejemplo de <ADRESS>:

Sintaxis básica:

<ABBR> Aqui pondriamos el termino abreviado </ABBR>

Esto es lo que hace el código anterior, en naranja el texto abreviado:

Sr.

EJEMPLO:

Ejemplo de <ADRESS>:

Sintaxis básica:

<ACRONYM> Aqui pondriamos el termino abreviado </ACRONYM>

Esto es lo que hace el código anterior, en naranja el texto citado:

ONU

PROPIEDADES:

Las etiquetas <ABBR> y <ACRONYM> no admiten propiedades especificas, pero una de las propiedades comunes a cualquier etiqueta la dotará de un aspecto visual diferente, esta es la propiedad TITLE.

  • TITLE: Con esta propiedad podremos definir el significado de la Abreviatura o del Acrónimo, el cual será visible en pantalla cuando mantengamos el ratón sobre la misma. Abajo vemos los ejemplos propuestos arriba aplicandoles la propiedad TITLE.

    • Ej. Ejemplo de <ABBR> con titulo, para ver la aplicación deja el ratón sobre la Abreviatura.

      Sr.

    • Ej. Ejemplo de <ACRONYM> con titulo, para ver la aplicación deja el ratón sobre el Acrónimo.

      ONU

    Como vemos en ambos ejemplos, no hay diferencias visuales, la diferencia es semantica, de cara al usuario no importa, está orientado a temas de busqueda.
